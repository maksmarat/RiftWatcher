# CCG Bot

CCG Bot - это Discord-бот для отслеживания матчей League of Legends у выбранной группы игроков. Он синхронизирует историю игр через Riot API, сохраняет сырые match payloads и производную статистику в SQLite и предоставляет slash-команды для анализа матчей, поиска общих игр, проверки текущего статуса, рекордов, статистики базы данных и автоматических уведомлений после матчей.

## Обзор

Проект построен вокруг трех идей:

- хранить локальную и удобную для запросов историю матчей отслеживаемых игроков
- обогащать данные Riot практичной аналитикой для Discord
- автоматизировать рутинные задачи сервера, включая уведомления о матчах и управление списком игроков

Бот использует slash-команды на базе `discord.py`, легковесный слой хранения на SQLite и фоновый watcher, который держит недавние матчи в актуальном состоянии.

## Возможности

### Аналитика матчей

- `/lolstats` синхронизирует недавние матчи игрока и показывает историю с пагинацией
- доступны фильтры очередей для solo queue, flex, arena, normal games и ARAM
- доступны фильтры по результату: победы, поражения или все матчи
- при наличии рядом с историей матчей показывается текущий solo queue и flex ранг игрока
- `/matchdetails` раскрывает выбранный матч в подробный разбор синей и красной команды с KDA, DPM, CS, золотом, названием очереди и длительностью
- в деталях матча теперь показываются общие места по матчу вроде `#1`, `#2`, а также бейджи `MVP` и `ACE` прямо в списке игроков
- MVP-оценка учитывает роль игрока и комбинирует боевую эффективность, экономику, обзор, полезность, выживаемость, игру по объектам и playmaking
- для рейтинговых матчей в деталях также показывается текущий ранг участников в соответствующей очереди
- `/records` считает лучшие сохраненные перформансы среди отслеживаемых игроков
- `/dbstats` показывает сводку по локальному датасету: игроков, матчей, сохраненных строк статистики, общее игровое время, среднюю длину матча и общее число question-mark пингов

### Поиск и сравнение

- `/search` поддерживает поиск по чемпиону в сохраненных матчах игрока или поиск общих матчей для одного игрока и еще до четырех Riot ID
- поиск по чемпиону использует autocomplete из загруженного набора чемпионов Riot
- результаты поиска поддерживают пагинацию и подсвечивают найденных игроков прямо в составе команды
- поиск общих матчей также возвращает компактную сводку по общему винрейту группы

### Отслеживание и уведомления

- отслеживаемые игроки автоматически опрашиваются в фоне каждую минуту
- watcher делает первичную синхронизацию, чтобы не отправлять старые матчи как новые уведомления
- новые матчи сохраняются один раз, даже если в одной игре было несколько отслеживаемых игроков
- уведомления группируются по матчу и отправляются только один раз в настроенный Discord-канал
- embed-уведомления о матчах теперь показывают те же глобальные места и бейджи `MVP` или `ACE`, что и `/matchdetails`
- для рейтинговых очередей уведомления включают название очереди и текущий ранговый текст участников
- данные ranked entries кратковременно кэшируются, чтобы уменьшить число повторных запросов к Riot API во время команд и фоновой синхронизации

### Утилиты сервера

- `/playerslol` показывает текущий список отслеживаемых игроков
- `/addplayer` и `/removeplayer` управляют списком отслеживаемых игроков
- `/allowuser` и `/disallowuser` управляют локальным whitelist для администрирования списка игроков
- `/setchannel` сохраняет Discord-канал для автоматических матч-уведомлений
- `/whoingame` проверяет, кто из отслеживаемых игроков сейчас находится в активной игре
- `/ping` и `/helplol` дают проверку доступности и встроенную справку по командам
- `/5stars` запускает мини-игру на пять игроков со случайной раздачей ролей и одной нелюбимой ролью на пользователя
- `/5stars` теперь не позволяет двум игрокам одновременно выбрать саппорта как нелюбимую роль в одной сессии

## Команды

Все аргументы Riot-игроков используют формат `gameName#tagLine`.

| Команда | Описание |
| --- | --- |
| `/lolstats nickname [count] [queue] [result]` | Синхронизирует недавние матчи игрока и показывает историю с пагинацией и опциональными фильтрами по очереди и результату. |
| `/matchdetails nickname match_number` | Показывает подробный разбор матча по нумерации из `/lolstats`, включая глобальные MVP-метки по участникам. |
| `/search nickname [character_name] [nickname_2] [nickname_3] [nickname_4] [nickname_5]` | Ищет в сохраненных матчах игрока либо по чемпиону, либо по общим играм с другими игроками. |
| `/whoingame` | Проверяет, кто из отслеживаемых игроков сейчас находится в активном матче. |
| `/records` | Показывает лучшие сохраненные рекорды среди отслеживаемых игроков. |
| `/dbstats` | Показывает сводную статистику по локальной SQLite-базе. |
| `/playerslol` | Показывает текущий список отслеживаемых игроков. |
| `/addplayer nickname` | Добавляет игрока в отслеживание. |
| `/removeplayer nickname` | Удаляет игрока из отслеживания. |
| `/allowuser user` | Разрешает Discord-пользователю управлять списком отслеживаемых игроков. |
| `/disallowuser user` | Убирает у Discord-пользователя доступ к управлению списком игроков. |
| `/setchannel` | Назначает текущий канал каналом для уведомлений о матчах. |
| `/5stars` | Запускает мини-игру с раздачей ролей для пяти игроков. |
| `/ping` | Проверяет, отвечает ли бот. |
| `/helplol` | Показывает встроенный embed со справкой по командам. |

## Примечания по командам

- Если не указывать `count` в `/lolstats`, бот использует все матчи этого игрока, которые уже сохранены в локальной базе.
- `/search` принимает либо `character_name`, либо дополнительные никнеймы игроков, но не оба варианта одновременно.
- `/matchdetails` использует ту же нумерацию матчей, что и `/lolstats`, где `1` - самый новый матч.
- В `/5stars` роль саппорта можно выбрать как нелюбимую только один раз на одну лобби-сессию.

## Архитектура

Кодовая база разделена на несколько небольших и понятных слоев:

- `bot.py` инициализирует Discord-клиент, Riot API клиент, подключение к базе, данные по чемпионам и расширения
- `cogs/` содержит пользовательские slash-команды
- `services/match_watcher.py` отвечает за фоновый polling, синхронизацию и отправку уведомлений
- `riot/riot_api.py` инкапсулирует запросы к Riot account, match, spectator и ranked-entry API
- `db/` содержит схему, SQLite-обертку и repository-слой
- `utils/` содержит файловые хелперы, общие утилиты форматирования и фильтрации, определения рекордов и роль-ориентированный движок MVP-оценки
- `data/` хранит локальную SQLite-базу и файловую конфигурацию

### Модель хранения

| Таблица | Назначение |
| --- | --- |
| `players` | Хранит известные Riot-идентичности по `puuid`. |
| `matches` | Хранит сохраненные матчи, метаданные очередей, тайминги и сырой Riot JSON payload. |
| `player_match_stats` | Хранит производные метрики по отслеживаемым участникам матчей. |

### Файловые данные

- `data/players.txt` хранит отслеживаемые `puuid`
- `data/allowed_users.txt` хранит Discord whitelist для команд управления игроками
- `data/config.json` хранит runtime-конфигурацию, например `notify_channel_id`
- `data/riftwatcher.db` хранит SQLite-базу, используемую ботом

## Технологии

- Python 3.11+
- `discord.py`
- `aiohttp`
- `aiosqlite`
- `python-dotenv`
- SQLite
- Riot Games API

## Быстрый старт

### 1. Установить зависимости

```bash
pip install discord.py aiohttp aiosqlite python-dotenv
```

### 2. Создать `.env`

```env
DISCORD_TOKEN=your_discord_bot_token
RIOT_API_KEY=your_riot_api_key
```

### 3. Запустить бота

```bash
python bot.py
```

При запуске бот:

- подключится к Discord
- инициализирует SQLite-схему из `db/schema.sql`
- скачает метаданные чемпионов из Data Dragon
- загрузит все cogs с командами и фоновый watcher

## Что демонстрирует проект

С точки зрения инженерии и портфолио проект показывает:

- разработку асинхронного Python-приложения
- создание Discord slash-команд на `discord.py`
- интеграцию с Riot API для account, match, spectator и ranked data
- локальное хранение данных в SQLite и repository-подход к доступу к данным
- фоновый polling и fan-out уведомлений
- вычисление производной аналитики на основе сырых данных стороннего API
- роль-ориентированную MVP-оценку на основе игровых метрик одного матча
